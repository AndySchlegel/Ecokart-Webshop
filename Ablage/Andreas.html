Key Architectural Decisions

This architecture leverages AWS managed services to minimize operational overhead while maximizing scalability and reliability. The serverless approach with Fargate eliminates the need for EC2 instance management, patching, and capacity planning. Aurora Serverless v2 provides automatic database scaling based on workload, ensuring cost efficiency during low-traffic periods while handling peak loads seamlessly.

Network Flow & Routing

Traffic flows from CloudFront to the public ALB in the public subnets. The ALB routes requests to Fargate tasks running in private subnets. Database connections are restricted to the database subnet group with no direct internet access. All outbound internet traffic from private subnets routes through NAT Gateways for security and compliance.

Security Architecture

Security Groups Configuration

ALB Security Group (sg-alb-public)

Ingress: 443/tcp from 0.0.0.0/0 HTTPS from Internet
Ingress: 80/tcp from 0.0.0.0/0 HTTP redirect to HTTPS
Egress: 8080/tcp to sg-fargate To Fargate tasks

Fargate Tasks Security Group (sg-fargate)

Ingress: 8080/tcp from sg-alb-public From public ALB
Ingress: 8080/tcp from sg-alb-internal From internal ALB
Egress: 5432/tcp to sg-aurora To Aurora PostgreSQL
Egress: 6379/tcp to sg-elasticache To ElastiCache Redis
Egress: 443/tcp to 0.0.0.0/0 External API calls via NAT

Aurora Database Security Group (sg-aurora)

Ingress: 5432/tcp from sg-fargate From Fargate tasks only
Ingress: 5432/tcp from sg-bastion Admin access (restricted)

ElastiCache Security Group (sg-elasticache)

Ingress: 6379/tcp from sg-fargate From Fargate tasks only

Additional Security Measures

  • AWS WAF on CloudFront for DDoS protection and SQL injection prevention
  • All data encrypted at rest using AWS KMS customer-managed keys
  • TLS 1.2+ enforcement for all data in transit
  • IAM roles with least-privilege access for all services
  • AWS Secrets Manager for database credentials and API keys
  • VPC Flow Logs enabled for network monitoring
  • AWS CloudTrail for API audit logging
  • Multi-factor authentication required for AWS console access

Service Discovery & Load Balancing

AWS Cloud Map provides service discovery for internal service-to-service communication. Each service registers with Cloud Map on startup, enabling dynamic endpoint resolution. The Application Load Balancer uses path-based routing to direct traffic to appropriate service target groups.
